附錄 C：建議學習資源

讀完前 18 章，你已具備把 Python 寫得「像 Python」而非「用 Python 寫 Java」的底氣。但語言與生態系演進快速，真正的進階來自持續閱讀一手資料。這份指南為資深 Java 工程師量身挑選：跳過入門教學的水分，直指官方規格、深度書籍與高密度社群。每項資源都標明「為何值得 / 對應本書哪幾章」，方便你按需取用。

C.1 官方文件（docs.python.org）

官方文件的品質遠勝多數第三方教學，且永遠與直譯器版本同步。對於習慣讀 JLS（Java Language Specification）與 JavaDoc 的你，這裡是最自然的起點。

文件 網址 為何值得 / 對應章節
The Python Tutorial https://docs.python.org/3/tutorial/ 官方導覽，半天可通讀。CP 值最高的一份。對應第 2–6 章
Language Reference https://docs.python.org/3/reference/ 等同 JLS：name binding、資料模型（data model）、執行模型。讀 datamodel 一章理解魔術方法。對應第 2、7、8、10 章
Library Reference https://docs.python.org/3/library/ 標準函式庫聖經。collections、itertools、functools、pathlib、asyncio 都在此。對應第 4、6、14、15 章
Python HOWTOs https://docs.python.org/3/howto/ 主題式深入文（logging、sorting、descriptor、regex、enum）。Descriptor HOWTO 必讀。對應第 10、12、15 章
What's New in Python https://docs.python.org/3/whatsnew/ 每版變更總覽。建議從 3.10 讀起：3.10 的 pattern matching（match/case）、3.12 的型別參數語法（PEP 695）、3.13 起的 free-threading（PEP 703）。對應第 1、9、14 章

重點 PEP（peps.python.org）

PEP（Python Enhancement Proposal）就像 Java 的 JEP，是語言設計決策的一手依據。在終端機輸入 import this 可隨時調出 PEP 20「Python 之禪」：

python 
>>> import this
The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
...
PEP 主題 對應章節
PEP 8 程式碼風格指南（命名、排版） 第 1、5 章。註：實務上交給 ruff/black 自動執行，不需手背
PEP 20 The Zen of Python 第 5 章
PEP 257 Docstring 慣例 第 3 章（docstring 慣例）、第 13 章（doctest）
PEP 484 型別提示（Type Hints）的奠基文件 第 9 章
PEP 703 移除 GIL，邁向 free-threading 第 14 章

小提醒：PEP 8 是「人讀的規範」，但今日的執行者是工具。ruff 與 black 會自動套用大半規則，你只需 ruff check 與 ruff format（詳見第 1 章），把腦力留給命名與架構。

C.3 線上教學與部落格

  • Real Python（https://realpython.com/）：品質最穩定的英文教學站，主題涵蓋全面，多數深度文有付費牆但免費內容已足夠。搭配第 13–17 章的實務主題閱讀。
  • 官方 Tutorial（見 C.1）：仍是最佳免費起點，不要跳過。
  • Raymond Hettinger 的演講：經典 PyCon 講題 "Transforming Code into Beautiful, Idiomatic Python"（YouTube 可搜尋）示範如何把 C/Java 式迴圈改寫成 Pythonic 寫法。對應第 5、6 章。他的 "Beyond PEP 8" 也值得一看。
  • mCoding（James Murphy，YouTube）：短而硬核，講 CPython 內部、型別、慣用法。對應第 2、9、10 章。
  • ArjanCodes（YouTube）：物件導向設計、SOLID、設計模式在 Python 的呈現，最貼近 Java 工程師的思考。對應第 7、8、11 章。

下面這類「把命令式迴圈 Pythonic 化」正是 Hettinger 反覆強調的，以 itertools 為例：

python 
>>> import itertools
>>> list(itertools.pairwise([1, 2, 3, 4]))   # 相鄰配對，免手寫索引
[(1, 2), (2, 3), (3, 4)]
>>> [(k, list(g)) for k, g in itertools.groupby("aaabbc")]
[('a', ['a', 'a', 'a']), ('b', ['b', 'b']), ('c', ['c'])]

C.7 給 Java 工程師的閱讀路線圖

把上述資源與本書 18 章串成一條 4–8 週的路徑。每階段先讀本書章節建立骨架，再用外部資源補肉。

第 1 週：地基與語感（對應第 1–5 章）

  1. 通讀官方 Tutorial（半天到一天）。
  2. 本書第 1–5 章：工具鏈、語法、函式、資料結構、Pythonic 慣用法。
  3. 開始 Exercism Python track，讓 review 糾正你的 Java 腔。
  4. 看 Hettinger 的 "Transforming Code into Beautiful, Idiomatic Python"。

第 2–3 週：物件模型與抽象（對應第 6–10 章）

  1. 本書第 6–10 章：迭代器、類別、繼承與協定、型別提示、裝飾器與元程式設計。
  2. 同步讀 Fluent Python 對應章節（資料模型、魔術方法、descriptor）。
  3. 愛型別的話加讀 Robust Python，把 Protocol 與 enum 內化。
  4. 閱讀 Descriptor HOWTO 與 PEP 484。

下面是貫穿第 7–9 章的核心對照——以不可變值物件為例，@dataclass(frozen=True, slots=True) 對應 Java 的 record（同樣自動產生 equals/hashCode）：

python 
>>> from dataclasses import dataclass
>>> @dataclass(frozen=True, slots=True)
... class Point:
...     x: int
...     y: int
...
>>> p = Point(1, 2)
>>> p
Point(x=1, y=2)
>>> hash(p) == hash(Point(1, 2))   # 自動產生 __hash__ 與 __eq__
True

對照現代型別提示寫法（list[int]、X | None、Callable 從 collections.abc）：

python 
>>> from collections.abc import Callable
>>> def first(items: list[int]) -> int | None:
...     return items[0] if items else None
...
>>> first([1, 2, 3])
1
>>> first([]) is None
True

第 4–5 週：工程實務（對應第 11–15 章）

  1. 本書第 11–15 章：模組與專案結構、例外、測試、並行、標準庫 I/O。
  2. 精讀 pytest 與 mypy 官方文件，把第 13 章心法落地。
  3. 用 LeetCode / Advent of Code 練資料結構與解析。
  4. 訂閱 PyCoder's Weekly 開始被動吸收。

第 6–8 週：應用與架構（對應第 16–18 章）

  1. 本書第 16–18 章：Web 開發、資料處理自動化、總整專案。
  2. 讀 FastAPI、SQLAlchemy 2.0、pydantic 官方文件。
  3. 讀 Architecture Patterns with Python，把你的 Spring/DDD 經驗對應到 Python。
  4. 用本書第 18 章 capstone 的 src layout + CI 模板，做一個自己的小專案收尾。

一句話總結：把官方 tutorial 從頭到尾通讀一遍，是你能做的 CP 值最高的投資——它便宜、權威、且永遠跟著直譯器更新。
